# BE-05: Prisma model + migration workflow

Add a model to `schema.prisma`, create a named migration, and apply it to the development database.

## Inputs

- `modelName` (required): PascalCase Prisma model name (e.g., `Article`)
- `fields` (required): array of `{name, type, modifiers?}` (e.g., `{name: "id", type: "String", modifiers: "@id @default(cuid())"}`)
- `relations` (optional): array of relation fields

## Outputs

Modifies/creates:

- `apps/backend/prisma/schema.prisma` — new model block appended
- `apps/backend/prisma/migrations/<timestamp>_<name>/migration.sql` — generated by Prisma CLI
- Confirms `postinstall` in `apps/backend/package.json` includes `prisma generate`

## Preconditions

- `DATABASE_URL` env var is set and the dev database is reachable
- `apps/backend/prisma/schema.prisma` exists with `datasource db` configured

## Workflow

1. Append the model block to `apps/backend/prisma/schema.prisma`:

```prisma
model <ModelName> {
  id        String   @id @default(cuid())
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  // TODO: add fields from inputs
}
```

2. Run the migration command:

```bash
# Creates migration file AND applies it to dev database
pnpm -C apps/backend prisma migrate dev --name <snake_case_model_name>
```

3. Verify Prisma Client is regenerated: `pnpm -C apps/backend prisma generate`
4. Confirm `apps/backend/package.json` `postinstall` includes `prisma generate`; if missing, add:

```json
"postinstall": "prisma generate"
```

## Commands reference

```bash
# Development — creates migration + applies
pnpm -C apps/backend prisma migrate dev --name add_article

# Production/CI — applies pending migrations only (no interactive prompts)
pnpm -C apps/backend prisma migrate deploy
```

## Error conditions

- `E_MIGRATION_CONFLICT`: History divergence (`migrate dev` warns about modified migrations) → run `prisma migrate reset` in dev only (WARNING: destroys dev data)
- `E_DB_CONNECTION`: `DATABASE_URL` unreachable → verify Docker is running (`docker compose up -d db`) and `.env` is loaded
